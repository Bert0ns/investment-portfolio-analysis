# PWA Offline Mode & Mobile Native Feel

## Problem Statement

How might we transform Capital Lens into an installable, standalone application that works entirely offline, removes browser UI clutter, and feels native on mobile devices?

## Recommended Direction

**The "Super PWA"**. We will implement a service worker via `@serwist/next` for aggressive offline caching of the static export. We will configure a web app manifest for `standalone` display mode to remove browser chrome, and integrate the Web Share Target API to allow seamless CSV imports directly from the mobile OS share sheet. We will defer true native wrappers (Tauri/Capacitor) until PWA limitations become an actual user pain point.

## Key Assumptions to Validate

- [ ] **Discoverability:** Users know how to (or can be easily prompted to) install a PWA. _Test: Implement a custom "Install App" prompt in the UI._
- [ ] **Storage Persistence:** IndexedDB is stable enough for long-term portfolio storage when installed to the home screen. _Test: Monitor storage eviction behavior, particularly on iOS._
- [ ] **Next.js App Router Compatibility:** We can reliably cache all necessary chunks and assets generated by the App Router for true offline capability. _Test: Airplane mode verification of all routes._

## MVP Scope

- **In:** `@serwist/next` integration for offline caching of all static assets.
- **In:** `manifest.json` configured for `standalone` mode, themed colors, and high-res icons.
- **In:** In-app "Install" button/prompt that triggers the beforeinstallprompt event.
- **In:** Web Share Target API integration routing to the global dropzone logic.
- **Out:** Push notifications, background sync, Tauri/Capacitor native wrappers, App Store deployments.

## Not Doing (and Why)

- **Tauri / Capacitor wrappers:** The maintenance overhead of native build pipelines (Xcode/Android Studio) is too high for current needs. The PWA gets us the desired "no browser chrome" UI today.
- **File System Access API:** Keeping IndexedDB as the primary storage maintains parity with the current web version and avoids complex permission flows for MVP.
